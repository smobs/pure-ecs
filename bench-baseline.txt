Baseline benchmarks for pure-ecs (PR perf/overhaul, before any further optimisation).
Backend: purs-backend-es 1.4.3 (optimising). Hardware: linux x64. Timer: Effect.Now (ms).
Iterations: 5 per scenario, simple arithmetic mean. Warm-up run discarded.

spawn 1000 entities (3 components each): 23 ms (avg of 5, sink=3)
query+update (pos, vel) over 1000 entities: 38 ms (avg of 5, sink=4)
10 movement ticks over 1000 entities: 354 ms (avg of 5, sink=4)

Notes:
  * `sink` is `world.structuralVersion` written into a Ref so the optimiser
    cannot dead-code-eliminate the scenario work.
  * Scenarios live in bench/src/. Run with `npm run bench`.
  * Numbers are noisy at this magnitude; expect +/-10% run to run. Trust ratios,
    not absolutes.
